# 概览 Tab PRD

概览 Tab 分为四个区域，按重要性排序：

1. **主信息区**（圆环饼图）— 核心指标一目了然
2. **CPU 使用率详情区** — 系统/用户/闲置占比
3. **系统状态区** — 温度、风扇、磁盘
4. **核心使用率区** — 性能核心 + 能效核心分组

---

## 1. 主信息区（圆环饼图）⭐ 最高优先级

展示三个圆环饼图，水平排列：

```
    ╭───╮          ╭───╮          ╭───╮
   ╱     ╲        ╱     ╲        ╱     ╲
  │  62%  │      │  41%  │      │  59%  │
   ╲     ╱        ╲     ╱        ╲     ╱
    ╰───╯          ╰───╯          ╰───╯
     CPU            GPU            MEM
```

| 指标 | 显示形式 | 交互 |
|------|---------|------|
| CPU 总使用率 | 圆环饼图 + 中心百分比 + 下方 label | 鼠标悬浮显示 System / User 占比 |
| GPU 利用率 | 圆环饼图 + 中心百分比 + 下方 label | 无悬浮 |
| 内存使用率 | 圆环饼图 + 中心百分比 + 下方 label | 无悬浮 |

CPU 悬浮提示示例：

```
┌─────────────────┐
│  System   48%   │
│  User     14%   │
└─────────────────┘
```

## 3. 系统状态区（文字显示） 

以文字形式展示以下信息：

```
🌡️ 温度    43℃
🌀 风扇    1001 RPM
💾 磁盘    可用 164 GB / 共 245 GB 
```

| 指标 | 显示内容 | 说明 |
|------|---------|------|
| 温度 | CPU 温度 | 单位支持 ℃ / ℉ 切换 |
| 风扇 | 当前转速 RPM | 多风扇时显示最大值 |
| 磁盘 | 可用空间 / 总空间 | 始终显示 |

### ⚠️ 动态显示规则

| 数据获取状态 | 显示行为 |
|-------------|----------|
| 温度获取成功 | 正常显示温度值 |
| 温度获取失败 | **隐藏整行**，不显示 "N/A" |
| 风扇转速获取成功 | 正常显示 RPM 值 |
| 风扇转速获取失败 | **隐藏整行**，不显示 "N/A" |
| 磁盘信息 | 始终显示（FileManager API 稳定可用） |

> 💡 设计理念：避免显示无意义的 "N/A"，保持界面简洁。

说明：

- 核心列表固定高度，超出可滚动查看
- 性能核心组和能效核心组分别显示，便于用户区分
- 单核负载为非首要关注信息，故放置于最下方
